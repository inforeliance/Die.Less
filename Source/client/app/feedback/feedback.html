App Feedback


The MedCheck project uses GitHubissues to report and track issues, feature enhancements, and new features. Be sure to be signed-up at GitHub and logged-in before proceeding.

We encourage your feedback and suggestions to improve this application. Feature suggestions and discussions are welcome. You may also propose changes by submitting a pull request. You don't need to install any software to suggest a change.

If you know the area you want to contribute to MedCheck, this is great, but if you are looking to make an initial contribution just ping our one of our developers on GitHub to tell us about your desire to work on a particular problem. For more complicated tasks, the best approach is also to bring that to the attention of the MedCheck developers, so they can give you some guidance on how best to tackle a particular problem, discuss implementation ideas and the semantics or scope of the proposed change.

Cloning the code base

To work on the MedCheck code base, you should be proficient enough in git and you should ideally have an account on Github to be able to create pull requests with your changes. If you have git installed on your machine, you should be able to clone the MedCheck repository with the following command:

git clone git://github.com/inforeliance/medcheck-core.git

Make sure you configure Git appropriately with the same email that you registered with on Github:

git config --global user.email "YOUR EMAIL"

You can verify these are configured appropriately by running:

git config --global user.email "YOUR EMAIL"

git config --list

Working on the code base


To run the project ensure you have the following npm repositories updated:
  • NodeJs
  • Bower
  • MongoDB
MedCheck uses Grunt for JavaScript task runner management, navigate to the /Source directory to build and run the project with the following commands:

Pull in the necessary repositories:

$npm update
$bower update

Run MedCheck:

$grunt serve

Test MedCheck:

$grunt test

Build MedCheck:

$grunt serve:build

Creating a pull request

Once you are satisfied with your changes:

  • commit your changes in your local branch
  • push your changes to your remote branch on Github
  • send us a pull requests
